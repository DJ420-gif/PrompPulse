What the Compliance API Does

The Anthropic Compliance API gives organisations programmatic access to audit logs across their Claude Platform deployment. It tracks two categories of events. Admin and system activities — adding or removing workspace members, creating or rotating API keys, modifying system configurations and any administrative action taken on the Claude Platform account. Resource activities — file creation, file deletion and resource-level operations performed by Claude agents or users within the workspace. The API does not log model inference interactions — the content of conversations between users and Claude is not captured in audit logs. This distinction matters for legal and privacy reasons: organisations can audit who did what at the system and resource level without Anthropic holding records of conversation content. For regulated organisations already using Claude Enterprise the Compliance API allows consolidation of multiple workspace audit feeds under a single parent organisation — giving compliance teams a unified view across all their Claude deployments rather than requiring manual aggregation across multiple accounts.

02

Why Regulated Industries Need This Now

Finance, healthcare and legal organisations face a structural compliance requirement that has blocked meaningful adoption of AI tools: you cannot deploy a system that takes actions — creating files, calling APIs, modifying records — in a regulated environment without an immutable audit trail of what the system did and when. This is not optional. SOX, HIPAA, and legal professional conduct rules all require audit trails for systems handling sensitive operations. Before the Compliance API, Claude could be used for reading and analysis tasks without issue but deploying Claude agents with write permissions in regulated environments required building custom audit infrastructure around the Claude API — a significant engineering investment that most compliance teams would not approve. The Compliance API removes that blocker. Organisations can now prove to auditors, regulators and internal compliance functions that every admin action and resource modification by Claude or Claude users is captured in a tamper-evident log accessible via API and exportable to their existing SIEM and compliance tooling.

03

How to Integrate the Compliance API

The Compliance API is available to organisations already using Claude Enterprise via the Anthropic admin console and API. The integration follows Anthropic's standard API authentication model — your existing API key authenticates to the compliance endpoint. Event streams are paginated and filterable by event type, timestamp range, user and resource. For SIEM integration the recommended approach is polling the API on a scheduled interval and forwarding events to your existing log aggregation infrastructure — Splunk, Datadog, Elastic or equivalent. The event schema is documented in Anthropic's developer documentation and events are delivered in JSON format compatible with standard log processing pipelines. For organisations with multiple Claude workspaces the parent organisation consolidation feature eliminates the need to poll multiple endpoints separately — a single authenticated request returns events across all child workspaces.

The Competitive Position — Compliance vs OpenAI and Google

Microsoft Azure OpenAI Service has offered audit logging for enterprise deployments for over a year — the Compliance API brings Anthropic's enterprise offering to feature parity on this dimension. Google Cloud Vertex AI similarly provides audit logging for Gemini deployments via Google Cloud Audit Logs. What differentiates Anthropic's Compliance API is the explicit design for multi-workspace enterprise deployments with parent organisation consolidation — a capability that enterprise customers with multiple business unit deployments have been requesting. The compliance posture also signals Anthropic's commitment to regulated industry expansion. The company's stated $19 billion ARR trajectory requires significant enterprise revenue, and regulated industries represent the largest untapped enterprise opportunity for AI platforms that can demonstrate appropriate compliance infrastructure. The Compliance API is both a technical feature and a strategic market signal.

Frequently Asked Questions

What does the Anthropic Compliance API do?
Provides programmatic access to audit logs for Claude Platform deployments. Tracks admin activities like member management and API key creation, and resource activities like file creation and deletion. Does not log conversation content. Available to Claude Enterprise organisations.
Which industries need the Anthropic Compliance API?
Financial services organisations under SOX and banking regulations, healthcare organisations under HIPAA, legal firms under professional conduct rules, and any regulated enterprise deploying Claude agents with write permissions where an immutable audit trail is required by regulators.
How is the Anthropic Compliance API different from Claude Enterprise?
Claude Enterprise is the subscription tier. The Compliance API is a new feature within that tier — launched March 30 2026 — that adds programmatic audit log access. Organisations already on Claude Enterprise can enable the Compliance API without upgrading their subscription tier.
Does the Anthropic Compliance API log conversation content?
No — the Compliance API explicitly does not log model inference interactions or conversation content. It logs admin and system activities and resource-level operations. This preserves confidentiality of conversation content while providing the operational audit trail regulators require.
Anthropic Compliance API vs Azure OpenAI audit logs — which is better?
Azure OpenAI audit logging via Azure Monitor has broader SIEM integration ecosystem and more established compliance certification documentation. Anthropic's Compliance API differentiates on multi-workspace parent organisation consolidation. For organisations not already on Azure infrastructure Anthropic's offering is a viable enterprise alternative.
